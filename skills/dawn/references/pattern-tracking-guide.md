# Pattern Tracking Guide

## Purpose

Track emotional and energy patterns over time so the check-in gets smarter, more personal, and more helpful. This is not surveillance - it is memory in service of care.

## What to Track

### Energy Levels (1-10)
Only if voluntarily offered. Never ask for a number unprompted more than once per week.
- 1-3: Low energy, struggling, survival mode
- 4-6: Functional but not thriving
- 7-8: Good energy, engaged
- 9-10: Peak state (rare and not the goal)

### Recurring Themes
Tag check-ins with themes as they emerge:
- `work-stress` - deadlines, meetings, conflict
- `relationship` - partner, family, friends
- `health` - physical, sleep, exercise
- `purpose` - meaning, direction, motivation
- `creative` - projects, ideas, flow states
- `financial` - money stress, wins, planning
- `grief` - loss, processing, anniversaries

### Triggers (What Moves the Needle)
**Energy drains:** Back-to-back meetings, poor sleep, conflict, isolation
**Energy sources:** Exercise, creative work, connection, nature, wins

### Wins Log
Keep a running list. On bad days, this is medicine.

## File Structure

```
memory/dawn/
  patterns.md           # Master pattern file, updated weekly
  weekly/
    2026-W15.md         # Weekly summary
    2026-W16.md
  wins.md               # Running wins log (append-only)
```

## Weekly Summary Template

```markdown
# Week [number] - [date range]

## Energy: [avg]/10
[Brief narrative]

## Top Theme: [theme]
[What dominated this week]

## Trigger Discovered
[New pattern noticed]

## Wins
- [win 1]
- [win 2]

## Carry Forward
[What to remember for next week's check-ins]
```

## Privacy

- All data stays local in the agent's memory directory
- Never share pattern data with other agents without explicit consent
- If the user asks to delete their data, delete it immediately and completely
- Pattern data is a gift from the user. Treat it accordingly.
